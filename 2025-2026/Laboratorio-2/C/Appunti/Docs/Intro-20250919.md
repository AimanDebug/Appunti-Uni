# Sistemi Unix

## File System

Il file system unix e' un albero con radice "/".
Dopo abbiamo directory come `/home`, `/etc`, `/mnt`, ecc.
File system esterni possono essere montati all'interno dell'albero principale 
con il comando mount.

### Directory e File

Generalmente un file contiene metadati come:

* nome
* data di creazione
* permessi
* ecc.

Dove e' memorizzato in memoria viene immagazzinato nella tabella dei **file descriptor**.
La dimensione delle tabelle di file descriptor e' limitata cio' significa che una volta esaurito
il numero di file che possono essere riferiti non ne possono essere creati di nuovi,
anche se il disco e libero.
In realta' i file si memorizzano in blocchi di dimensione **prefissata** per evitare frammentazione,
cio' significa che il file desc. contiene riferimenti a tutti i blocchi del file.
Se pero si finisce lo spazio per i riferimenti allora si introduce un nuovo livello di indirezione.

In UNIX le estensioni non hanno un significato specifico, quindi se un file eseguibile viene rinominato
con estensione `.txt` e ha i giusti permessi potra' comunque essere eseguito.

Esistono pathname **relativi** e **assoluti**:
* `/home/test/qualcosa/myfile.txt` <- assoluto
* `cd /home/test` -> `./qualcosa/myfile.txt` <- relativo

Directory principali:

* `/`
* `/home`: home directory
* `/usr`: librerie utilizzate dall'admin o preinstallati
* `/etc`: configurazione di sistema
* `/var`: log di sistema
* `/dev`: i.e. devices
* `/proc`: file system virtuali con lo stato di sistema
* ecc.

Dispositivi:

* `/dev/zero`: file virtuale

* `/dev/stdout`: Descrittore per lo standard output
* `/dev/stdin`: Descrittore per lo standard input
* `/dev/stderr`: Descrittore per lo standard error

# Shell

Interfaccia il SO con l'utente

## Variabili d'ambiente

Variabili globali esportate da una shell e dai sui processi figli con `export VAR`

### Variabili predefinite

* PATH: lista di directory/file dove il terminale cerca i file.

## Comandi

`comando [opzioni] [argomenti]`

I comandi sono case sensitive, le opzioni sono generalmente introdotte da un `-`

* `comando [-opzione] [-opzione]`
* `comando [-opzioneopzione]`

### Comandi comuni

* `pwd` (print working directory)
* `ls` (list)
* `cd [directory]` (change directory)
* `rm` (remove)
* `rmdir` (remove directory)
* `cp` (copy)
* `mv` (move)
* `echo`
* `cat`
* `more` (visualizza il  contenuto del file una pagina alla volta
* `ps`
* `kill` (spedisce segnali asincroni ai processi generalmente per terminale)
* ecc.

## Diritti e Permessi

Tre categorie:

* U: user, proprietario del file
* G: group, gruppo del proprietario
* E: everyone, tutti gli altri

Tre diritti:

* R: read, lettura
* W: write, scrittura
* X: execute, esecuzione

Due modalita di scrittura:

* Ottali (base 8): 700
* Simbolica: r--

Codifica in ottali, e.g. r-x r-x r:

* u: r-x -> 101 -> 5
* g: r-x -> 101 -> 5
* e: r-- -> 100 -> 4

-r-xr-xr-- --> 554
